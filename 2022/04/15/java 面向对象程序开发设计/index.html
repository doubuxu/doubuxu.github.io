Java 面向对象程序开发设计

类的介绍

Java的特点是面向对象开发，现实世界中有对象，程序世界也有对象，二者相差很大，为了通过程序设计解决现实世界中的问题，要让程序中的对象尽可能地与现实世界中的相似以方便开发，维护程序。

在现实世界中，==类（class）即指一类性质相似的事物，而对象（object）指的是这类事物中具体的一个==。一个对象有状态和行为两大类特征，同一类的对象这两种特征相似；对应的，==代码中的对象也有状态（值）和行为（方法）两大特征。==

为了模拟现实世界中的对象，我们在程序中根据需求自定义一些类，在类中定义与需求相关的状态（变量）和行为（方法），利用这些类解决问题。

类就是一种==自定义的数据类型==

构造方法

构造方法是类中一种特殊的方法，用于初始化对象。构造方法的声明中同样也有修饰符，但没有返回类型（void也没有），命名必须和类名相同，这样编译器才会知道这是一个构造方法。如果类中没有构造方法，那么编译器会在字节码文件中自动加入一个构造方法，方法体为空。

构造方法也可以重载，使用关键字this（）

方法重载

当一个类中的好几个方法内容相似时（但按常理因为一些细小差异你还得一个一个写出来），为了方便，编译器允许这几种方法同名以便于记忆和使用。但方法的声明不能完全相同，一般参数不同，以便区分。

构造方法也可重载

static

在修饰成员变量时，可加入修饰符static，例如（private static int number）。如此定义的成员变量属于这个类，不像其他成员变量属于对象，而类只是一个规范。由这一个类定义出的所有对象的static变量是相同的，改变一个对象中static变量的值其余所有由该类创建的对象的number变量的值均会发生相同的改变。此外，与一般的成员变量不同，static变量可直接由类访问，例如（class.number=5），这一操作也会同时改变所有对象的number值。

类的继承

###继承的介绍

如果有两个类他们的结构相似，这个时候如果分别定义这两个类，不仅代码重复，也不利于后期维护。而Java语言支持类的继承。

继承反应类之间的关系，例如（A is a B),其中，A是一个子类，B是一个父类。A继承了B的成员变量和方法，在B的基础上，A可以继续扩展，丰富功能。也就是说，B是A的一个子集，A是B的一个扩展。

继承的定义

###子类继承

子类不能继承以下

  1. 父类的private属性，方法
  2. 父类的构造方法

###单继承

Java目前只支持单继承，一个子类只能有一个父类，一个父类可以有多个子类（目前指我学的水平，多继承靠接口实现）

super关键字

super关键字指向所在类的父类，引用父类的属性或方法
